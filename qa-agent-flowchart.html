Internal QA Agent Workflow Flowchart

Flowchart view of the current internal smoke testing process with Streamlit control, Selenium execution, reporting, email, and Jira integration.
Start

User Opens Streamlit Dashboard

User selects JIRA mode and enters a command such as navigation, form, performance, mobile, or full smoke run.

Control

Mode Selection and Trigger

The dashboard maps the command to the execution mode and calls the AI test runner.

Initialize

Agent Setup

The agent starts, loads the knowledge base, prepares report and screenshot folders, and initializes result containers.

Execution

Launch Browser and Run Smoke Tests

Chrome WebDriver starts and executes website accessibility, navigation, search, page, form, performance, and mobile checks based on mode.

Resilience

Handle Popups and Smart Locators

The agent closes cookie banners or modal popups and searches elements using CSS, XPath, text, ID, and name strategies.

Test Passed?

Result is evaluated for pass, fail, warning, or recovery through fallback logic.

Yes

Store Pass Result

Status, message, details, and timestamp are saved to the result set for reporting.

No

Capture Failure Evidence

The agent logs the issue, captures screenshots, stores the URL and failure details, and marks the test as failed.

Reporting

Generate Reports

The system generates the HTML report, prepares issue documents when defects exist, and summarizes total tests, pass rate, and issues found.

JIRA Enabled?

The workflow checks whether the run was started with Jira integration turned on.

Yes

Create or Update Jira Ticket

Issue details and generated evidence are sent to Jira so the failure becomes a tracked work item.

No

Skip Jira Step

The workflow continues without ticket creation while still preserving reports and alerts.

Notify

Send Email and Show Dashboard Output

The agent sends email notifications and the Streamlit dashboard shows execution summary, test details, reports, screenshots, and run history.

Business summary: The internal QA workflow starts from the Streamlit dashboard, executes smoke tests through Selenium, handles UI instability using smart locator logic, records evidence, generates reports, optionally creates Jira issues, and gives the team immediate visibility through email and dashboard output.
